# 2D背景与3D模型分层实施计划

> 目标：将 Photo-Wall 的视觉底层拆成「2D 背景层 + 透明 3D 模型层 + DOM UI 层」，让背景效果脱离 WebGL 场景限制，只保留「塔式 / 纸片」模型为 3D 表现。本文为实施方案与验收标准，不记录执行结果。

## 1. 目标

当前视觉体系中，背景增强效果曾挂在 R3F 场景内。这样会让背景表现受 3D 相机、WebGL 性能、透明混合和模型层级共同约束。

新的目标结构：

```text
BackgroundLayer
  2D 背景、光场、网格、纹理、主题氛围

Scene Canvas
  透明 WebGL Canvas，只渲染塔式 / 纸片模型

DOM UI
  Header / Footer / 菜单 / 页面内容

PageTransition
  路由切换遮罩，保持最高层级
```

## 2. 第一阶段范围

第一阶段只做架构拆层：

- 新增 `BackgroundLayer`；
- 将 `BackgroundLayer` 接入 `AppLayout`；
- 让 R3F Canvas 透明；
- 从塔式页移除 3D 背景 `FloatingLines`；
- 页面容器背景改为透明；
- 保留 `FloatingLines.jsx` 作为备用，不立即删除；
- 同步代码地图。

第一阶段暂不做：

- 复杂 2D canvas 粒子；
- 视频背景；
- 图片主色自动提取；
- 新增依赖；
- 大规模主题数据结构调整；
- 移除 Footer 模型切换。

## 4. 分层规则

建议层级：

```text
BackgroundLayer: z-index 0
Scene Canvas: z-index 1
Page DOM / View: z-index 2 或正常文档流
Footer: z-index 1000
Header: z-index 1100
Header menu overlay: z-index 1200+
PageTransition: 最高层
```

`BackgroundLayer` 必须满足：

```text
position: fixed
inset: 0
pointer-events: none
aria-hidden="true"
```

## 5. 背景设计原则

第一版背景使用 CSS 完成：

- 底色使用深炭黑和暖灰黑；
- 使用主题强调色生成柔和光场；
- 使用轻量网格和纹理增强空间感；
- `/`、`/paper`、`/directory` 根据路由切换不同背景倾向；
- 动效只做慢速、低频漂移；
- 支持 `prefers-reduced-motion`。

避免：

- 大量 DOM 粒子；
- 高频 JS 动画；
- 过重 `filter` 动画；
- 全屏动态 `backdrop-filter`；
- 单一大面积紫蓝渐变。

## 6. 主题联动

第一阶段不改 `themes.js` 数据结构。

背景组件内部维护轻量映射：

```text
default -> 金色 / 青色
fengjing-jianzhi -> 绿色 / 暖米色
fallback -> default
```

后续当主题系统稳定后，再考虑把背景强调色沉淀到主题注册表。

## 9. 第二阶段范围

第二阶段在不改动 3D 模型逻辑、不新增依赖的前提下，增强 `BackgroundLayer` 的视觉表现：

- 扩展主题调色板映射；
- 增加第三强调色与基础底色变量；
- 增加 `chroma`、`architecture`、`horizon` 等 CSS 层；
- 强化 `/`、`/paper`、`/directory` 的背景差异；
- 移动端降低结构线与纹理强度；
- `prefers-reduced-motion: reduce` 下关闭背景漂移动画。

第二阶段仍然不做：

- 图片主色自动提取；
- Canvas 2D 粒子系统；
- 视频背景；
- WebGL 后处理；
- 主题数据结构变更。

## 11. 第三阶段：React Bits 风格 2D 漂浮线条

第三阶段在维持“背景归 2D、模型归 R3F”的分层前提下，将 React Bits `Floating Lines` 的视觉语言落到 `BackgroundLayer` 内部，而不是重新挂回 `src/three/scenes/FloatingLines.jsx`。

新增模块：

```text
src/components/background/FloatingLines2D.jsx
docs/背景系统/Floating-Lines-2D背景层说明.md
```

接入原则：

- 只在 `/` 塔式照片墙页启用；
- 复用已有 `three`，不新增依赖；
- 使用独立 `WebGLRenderer` 渲染全屏 shader 平面；
- 由 `BackgroundLayer` 传入主题强调色生成线条渐变；
- 指针交互监听 `window`，不改变背景层 `pointer-events:none`；
- 降低动态偏好下不启动动态线条；
- 保留 `src/three/scenes/FloatingLines.jsx` 作为 3D 对照和回滚方案。

验收标准：

- [ ] 首页存在 React Bits 风格三层漂浮线条；
- [ ] 线条位于照片墙背后，不遮挡照片柱体和 Banner；
- [ ] 主题切换后线条渐变跟随当前主题；
- [ ] 鼠标移动时存在轻微弯曲和视差；
- [ ] `/paper` 与 `/directory` 不挂载动态线条；
- [ ] `prefers-reduced-motion: reduce` 下不启动渲染循环；
- [ ] 构建通过。
