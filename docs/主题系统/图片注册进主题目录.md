# 图片注册进主题目录

> 本文说明当前项目把图片注册为主题，并在「主题目录」页面中展示和切换的方法。

## 1. 注册链路总览

当前项目采用构建期静态主题注册方式：

```text
public/photos/<主题id>/图片文件
        ↓
src/data/themes.js 注册 presetThemes
        ↓
src/context/ThemeContext.jsx 暴露 themes / activeTheme / setActiveTheme
        ↓
src/components/theme/ThemeDirectory.jsx 渲染主题目录并切换主题
        ↓
src/app/page.jsx 与 src/app/paper/page.jsx 读取 activeTheme.images 生成拼贴纹理
```

核心原则：新增主题只需要放置图片资源，并在 `src/data/themes.js` 追加一条主题记录；页面和主题目录会自动读取注册表。

## 2. 图片资源目录规则

主题图片统一放在：

```text
public/photos/<主题id>/
```

示例：

```text
public/photos/default/img1.webp
public/photos/default/img2.webp
public/photos/fengjing-jianzhi/风景剪纸_1.png
public/photos/fengjing-jianzhi/风景剪纸_2.png
```

约定：

- `<主题id>` 必须和 `src/data/themes.js` 中主题对象的 `id` 一致。
- 图片路径在代码中从站点根路径开始写，例如 `/photos/default/img1.webp`。
- 默认优先使用适合 Web 的图片格式，例如 `webp`；已有主题也支持 `png`。
- 同一主题内建议使用稳定、连续、易读的命名，便于用 `Array.from` 批量生成图片清单。

### 2.1 新增图片压缩规范

新增主题图片在注册进主题目录前，必须先压缩为 WebP 格式，再写入 `src/data/themes.js` 的 `cover` 与 `images` 路径。

规范要求：

- 主题目录中用于页面加载的图片优先使用 `.webp`，避免直接注册未压缩的 `.png`、`.jpg` 或 `.jpeg` 原图。
- 原始图片可以保留在本地素材目录或同目录中，但主题注册路径必须指向压缩后的 WebP 文件。
- WebP 文件名应与原图保持同一编号规则，例如 `风景剪纸_1.png` 对应 `风景剪纸_1.webp`。
- 批量转换时建议使用 `quality 82`、`webp:method=6` 作为默认压缩参数，在画质和体积之间保持平衡。
- 压缩完成后必须核对 WebP 数量、文件体积和至少一张图片的尺寸，确认输出文件有效后再更新主题注册。

示例：

```js
const fengjingJianzhiImages = Array.from({ length: 17 }, (_, i) => ({
    url: `/photos/fengjing-jianzhi/风景剪纸_${i + 1}.webp`,
}));
```

## 3. 主题注册位置

主题注册表位于：

```text
src/data/themes.js
```

导出的 `presetThemes` 是当前主题目录的数据源。每个主题对象结构如下：

```js
{
    id: 'fengjing-jianzhi',
    name: '风景剪纸',
    cover: '/photos/fengjing-jianzhi/风景剪纸_1.png',
    tags: ['PAPER-CUT', 'SCENERY', 'ILLUSTRATION'],
    images: fengjingJianzhiImages,
}
```

字段说明：

| 字段     | 必填 | 说明                                                    |
| -------- | ---- | ------------------------------------------------------- |
| `id`     | 是   | 主题唯一标识，同时对应 `public/photos/<主题id>/` 目录名 |
| `name`   | 是   | 主题目录中展示的名称                                    |
| `cover`  | 否   | 主题封面图；不填时组件可退回使用首张图片                |
| `tags`   | 否   | 主题标签，用于主题目录列表展示                          |
| `images` | 是   | 图片清单，格式为 `[{ url: '/photos/...' }]`             |

## 4. 默认主题与普通主题

默认主题图片清单单独放在：

```text
src/data/images.js
```

该文件导出 `default` 主题的 33 张图片：

```js
const images = [{ url: '/photos/default/img1.webp' }, { url: '/photos/default/img2.webp' }];

export default images;
```

然后在 `src/data/themes.js` 中复用：

```js
import images from './images';

export const presetThemes = [
    {
        id: 'default',
        name: '默认',
        cover: '/photos/default/img1.webp',
        tags: ['PHOTOGRAPHY', '3D SCENE', 'INTERACTIVE', 'WEBGL'],
        images,
    },
];
```

普通主题可以直接在 `themes.js` 中生成图片清单，例如当前「风景剪纸」主题：

```js
const fengjingJianzhiImages = Array.from({ length: 17 }, (_, i) => ({
    url: `/photos/fengjing-jianzhi/风景剪纸_${i + 1}.png`,
}));
```

## 5. 新增主题操作步骤

### 步骤 1：创建主题图片目录

在 `public/photos/` 下创建新目录：

```text
public/photos/<新主题id>/
```

示例：

```text
public/photos/city-night/
```

### 步骤 2：放入图片资源

示例：

```text
public/photos/city-night/city-night-1.webp
public/photos/city-night/city-night-2.webp
public/photos/city-night/city-night-3.webp
```

### 步骤 3：在 `src/data/themes.js` 生成图片清单

如果图片命名连续，推荐使用 `Array.from`：

```js
const cityNightImages = Array.from({ length: 12 }, (_, i) => ({
    url: `/photos/city-night/city-night-${i + 1}.webp`,
}));
```

### 步骤 4：追加主题记录

把主题对象追加到 `presetThemes`：

```js
export const presetThemes = [
    // existing themes...
    {
        id: 'city-night',
        name: '城市夜景',
        cover: '/photos/city-night/city-night-1.webp',
        tags: ['CITY', 'NIGHT', 'PHOTOGRAPHY'],
        images: cityNightImages,
    },
];
```

完成后，主题会自动出现在 `/directory` 的主题目录页中。

## 6. 运行时切换流程

主题状态中心位于：

```text
src/context/ThemeContext.jsx
```

关键逻辑：

- `themes` 来自 `presetThemes`。
- `activeId` 是当前主题 ID。
- `activeTheme` 通过 `themes.find((t) => t.id === activeId)` 派生。
- `setActiveTheme(id)` 会切换主题，并把当前主题 ID 写入 `localStorage`。

主题目录组件位于：

```text
src/components/theme/ThemeDirectory.jsx
```

点击主题项时会调用：

```js
setActiveTheme(id);
```

页面使用：

```js
const { activeTheme } = useTheme();
```

并把 `activeTheme.images` 传给 `useCollageTexture` 生成拼贴纹理，因此主题切换后塔式照片墙和纸片视图会随当前主题更新。

## 7. 注意事项

- `id` 不要重复，否则 `activeTheme` 只会匹配到第一项。
- `cover` 路径和 `images` 路径必须指向 `public` 下真实存在的文件。
- 如果主题图片数量较少，纸片视图只会取当前主题前 5 张图片。
- 当前 `themes.js` 中存在 `TEMP_THEMES` 临时测试主题，标记为动画测试用，后续测试完成后应删除。
- `src/three/scenes/Banner.jsx` 中的 `/banner.jpg` 是独立横幅资源，不属于主题图片注册链路。
